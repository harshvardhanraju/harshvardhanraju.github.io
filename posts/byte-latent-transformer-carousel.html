The Problem
Fixed tokenization allocates wrong
Predictable English
the quick fox
3 tokens. Each is entirely predictable given context. Same compute as hard text.
Arabic morpheme (same compute)
▁ ج م ي ل
5 tokens. Each character could be many continuations. But gets identical attention budget.
Typo (tokenizer fails)
teh
"teh" → tokenizer produces unfamiliar sequence. Model has no recovery path.
Root cause
Vocabulary built from frozen corpus snapshot. Boundaries don't reflect what's actually hard about this sequence right now.

Architecture
Three components, one principle
Local Encoder
Small transformer.
Reads raw bytes.
Forms patch vectors.
→
Latent Transformer
Large transformer.
Runs on patches only.
All reasoning here.
→
Local Decoder
Small transformer.
Reconstructs byte
probabilities.
The expensive quadratic attention only runs on patches — not individual bytes. Each patch represents a variable number of bytes, so the latent transformer's sequence length is much shorter than the raw byte count. This is how byte-level becomes compute-feasible.
Input
Raw bytes
Attention scope
Patches only
Vocabulary
None
Alphabet

The Key Mechanism
Entropy → patch size
Next-byte entropy across a sequence (higher = harder = smaller patch)
the ▁f جـمـيـ ل▁ 1234
Resulting patches:
the fox جـمـيـ ▁ 1234
■ large patch — low entropy, easy text ■ medium patch ■ single byte — high entropy, hard text

What's Still Open
Proven at 8B. Unknown at 70B+.
What BLT proved
✓ Parity
Byte-level and token-level are compute-competitive at 8B. This wasn't true a year ago.
Still unproven
? 70B+
Nobody has run the experiment at frontier scale. Local encoder overhead may bottleneck at very large sizes.
Clear advantages
Noise robustness (+8–15%), rare script handling (+12%), zero token tax, no OOV tokens, no vocabulary to maintain.
Engineering challenges
Efficient byte-level inference at scale is unsolved. Patch size tuning affects performance. Training infrastructure needs changes.
"Can byte-level models work?" is no longer the question. "When is it worth deploying?" is.
