Algorithm Landscape
4 Algorithms, All in Production
BPE
Greedy frequency-based merges. Start with characters, repeatedly merge the most frequent adjacent pair. Fast, deterministic, dominant.
Used by: GPT-4o · LLaMA 3 · Qwen · DeepSeek · Mistral
Unigram LM
Works in reverse: start over-complete, prune tokens whose removal hurts corpus likelihood least. Has a proper objective function.
Used by: Gemma · Gemini · T5 · PaLM · LLaMA 1/2
WordPiece
Merges pairs with highest PMI score: freq(AB) / (freq(A)·freq(B)). More linguistically coherent but largely legacy now.
Used by: BERT · DistilBERT (largely superseded)
SentencePiece
A framework, not an algorithm. Runs BPE or Unigram internally. Handles raw text without pre-tokenization — language-agnostic by design.
Used by: Google ecosystem · LLaMA 1/2 · many multilingual models

What Each Major Lab Changed
Different Bets, Same Direction
OpenAI
100K → 200K (o200k_base)
Doubled vocabulary for GPT-4o. Reduced CJK token tax by ~40%. Contains unreachable tokens — vocabulary artifacts discovered post-launch.
Meta
32K → 128K (complete rewrite)
LLaMA 2→3: scrapped SentencePiece for tiktoken-style BPE. 4× vocabulary jump added ~1B parameters before a single transformer layer. Locked unchanged across 3.1/3.2/3.3.
Google
Unigram LM at 256K
Largest in production. Added split-digit tokenization — each digit gets its own token — as a direct fix for arithmetic failure. Shared across Gemma and Gemini.
Qwen / DeepSeek
Extend, don't rebuild
Qwen starts from cl100k_base and adds 50K+ CJK tokens. DeepSeek uses 32K for code (domain is constrained) and 128K for general. Pragmatic reuse over reinvention.

Fixed Tokenization

ACL 2025 · The Bombshell Result
Optimal tokenization is NP-complete
"Tokenisation is NP-Complete" (arXiv:2412.15210) proved both problems are in the NP-complete complexity class: choosing the optimal vocabulary and finding the optimal encoding for a given vocabulary. Reduction from set cover. Unless P = NP, no polynomial-time algorithm exists.
Vocabulary selection
Choosing which merges to keep to minimize total tokens across a corpus is NP-complete. BPE is a greedy heuristic, not optimal.
Encoding problem
Given a fixed vocabulary, finding the segmentation that minimizes token count is also NP-complete. The fertility landscape is not smooth.
